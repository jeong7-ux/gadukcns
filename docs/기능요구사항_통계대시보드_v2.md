# 통계 대시보드(S-10) 기능요구사항 정의서 v2.0

> 작성일: 2026-07-12 · 대상 화면: **S-10 통계 대시보드** (FR-11 재정의·강화) · 범위: 대시보드 현황 전용
> 목표: 수집된 DB와 적용된 전 기능(주력분야 스코어링·고객사 우선순위·아카이브·AI요약·관심목록)을 종합하여, **한눈에 모든 입찰정보 현황**을 보여주는 커맨드센터로 강화.

---

## 1. 개요 / 목적

현행 S-10은 KPI 4종 + 차트 4개의 기본 대시보드다. 이를 **"열자마자 오늘 무엇을 해야 하고, 전체가 어떤 상태인지"** 를 한 화면에서 파악하는 **의사결정 대시보드**로 재정의한다.

- 대상 사용자: 경영진(exec)·전략기획(strategy)·사업관리(pm) — 역할별 관점 제공.
- 핵심 가치: **한눈(single-glance)** · **액션 지향(무엇을 해야 하나)** · **실시간(Realtime)** · **드릴다운(클릭 → 상세/목록)**.

## 2. 설계 원칙

1. **정보 계층화(F-패턴)**: 상단=핵심 KPI(밴드) → 중단=액션/현황 → 하단=심층 분석·추세.
2. **한 화면(above-the-fold)에 핵심**: KPI 밴드 + 마감임박 액션 + 주력분야 + 고객사 현황을 첫 스크롤 없이.
3. **모든 지표는 클릭 가능(드릴다운)** → 해당 S-04/S-07 목록으로 필터 이동.
4. **델타·추세 병기**: 절대값 + 전일/전주 대비 변화 + 스파크라인.
5. **디자인 토큰 준수**: 기존 색/상태 pill/D-day 3구간 재사용. 신규 색은 토큰화.

## 3. 화면 레이아웃 (영역 구성)

```
┌───────────────────────────────────────────────────────────────┐
│ [전역 필터] 기간 · 주력분야 · 고객사 카테고리 · 상태 · 새로고침·실시간  │  (DFR-10)
├───────────────────────────────────────────────────────────────┤
│ [KPI 밴드] 10 타일 (값 + 전일대비 델타 + 스파크라인)               │  (DFR-01)
├───────────────────────┬───────────────────────────────────────┤
│ 마감 임박 · 액션 보드    │ 주력분야 현황(공고·평균점수·진행중)          │  (DFR-02 / DFR-03)
│ (D0~3, 관심 진행상태)   │                                        │
├───────────────────────┼───────────────────────────────────────┤
```

## 4. 위젯별 기능요구사항 (DFR-01 ~ DFR-10)

### DFR-01 핵심 KPI 밴드 (10 타일)
- **목적**: 전체 상태를 숫자로 즉시 파악. 각 타일 = 값 + 전일 대비 델타(▲▼) + 미니 스파크라인.
- **타일**:
  1. 주력 관련 공고(score>0, 미아카이브)
  2. 진행중(status ongoing)
  3. 임박 마감(D0~3)
  4. 고득점(score≥5)
  5. **고객사 공고**(발주/수요=고객사)
  6. 관심목록(watchlist)
  7. 신규 24h(notice_dt ≤ 24h)
  8. 평균 점수(관련 공고)
  9. **총 추정가 규모**(Σ est_price, 억원)
  10. **AI 요약 커버리지**(ai_summary 有 / 관련 공고, %)
- **소스**: bids, clients, watchlist. 델타: 일일 스냅샷(DFR-11, 신규 테이블) 또는 notice_dt 근사.
- **상호작용**: 타일 클릭 → 해당 조건으로 S-04 필터 이동.

### DFR-02 마감 임박 · 액션 보드
- **목적**: "지금 처리해야 할 것". 첫 화면 우선 배치.
- **표시**:
  - **임박 마감 공고 리스트(D0~3)**: 공고명·발주기관·D-day pill·점수·고객사 여부. 마감 오름차순.
  - **관심목록 진행 현황**: 분석(none/in_progress/done)·제안(none/writing/done)·결정(review/join/drop) 카운트 + "미착수·진행중" 강조.
  - **오늘의 액션**: 오늘 마감(today) + 관심등록됐으나 미분석 공고 수.
- **소스**: bids(status/deadline_dt/client_name), watchlist(analysis_status/proposal_status/decision).
- **상호작용**: 항목 클릭 → S-06 상세 / S-07 관심목록.

### DFR-03 주력분야 현황
- **목적**: 정보시스템 감리·컨설팅 분야별 기회 분포.
- **표시**: 분야(정보시스템감리·ISP/정보화컨설팅·정보보안·SI/시스템구축·성과평가·유지보수·AI/데이터)별 **공고 수 · 평균 점수 · 진행중 비율**. 누적 막대 또는 그룹 막대.
- **소스**: bids.tags(키워드 룰 매칭) ↔ 분야 매핑, keyword_groups.
- **상호작용**: 분야 클릭 → 해당 키워드그룹으로 S-05 검색.

### DFR-04 고객사 현황
- **목적**: 회사 고객사(보건·의료 중심) 기반 재수주·영업 기회.
- **표시**:
  - **카테고리별 고객사 공고 수**(중앙부처·지자체·공공·의료·교육·금융) — 도넛/막대.
  - **고객사 Top N**(공고 많은 고객사) — 막대.
  - **재수주 기회**: 고객사 발주·진행중 공고 수, **보건·의료 집중률**(의료+보건복지 공고 / 전체 고객사 공고).
  - 신규 고객사 공고(24h) 배지.
- **소스**: clients(category/is_priority), bids(order_org/demand_org 매칭, status).
- **상호작용**: 고객사/카테고리 클릭 → S-04 필터(해당 기관) 이동.

### DFR-05 스코어링 · 품질 현황
- **목적**: 스코어링 결과 분포와 우량 공고 확인.
- **표시**:
  - **점수 분포 히스토그램**(0·1~4·5~9·10~14·15+).
  - **AI 점수 분포**(ai_score 구간).
  - **고득점 Top 10**(score desc) — 공고명·점수·AI·고객사 배지, 바로가기.
  - **AI 요약 커버리지 게이지**(요약 有/관련 공고).
- **소스**: bids(score/ai_score/ai_summary).
- **상호작용**: Top 항목 클릭 → S-06 상세.

### DFR-06 시계열 추세
- **목적**: 유입·마감 흐름 파악.
- **표시**:
  - **일자별 신규 공고**(최근 14~30일, notice_dt) — 라인.
  - **향후 마감 예정**(다음 14일, deadline_dt) — 막대(임박 색상).
  - (선택) 주간/월간 토글.
- **소스**: bids(notice_dt/deadline_dt).

### DFR-07 발주/수요기관 현황
- **목적**: 어디서 공고가 나오는가.
- **표시**: **발주기관 Top 8** · **수요기관 Top 8**(고객사는 강조), **고객사 vs 일반 비율** 도넛.
- **소스**: bids(order_org/demand_org), clients.
- **상호작용**: 기관 클릭 → S-04 필터 이동.

### DFR-08 AI 데일리 브리핑
- **목적**: 오늘의 AI 종합 요약.
- **표시**: `daily_brief.summary`(당일) + `top_bids`(상위 공고 링크). 날짜 선택 가능.
- **소스**: daily_brief.

### DFR-09 데이터 · 파이프라인 상태
- **목적**: 데이터 신선도·건전성 확인(신뢰).
- **표시**: **마지막 수집 시각**(collect_cursor.last_reg_dt), 총 공고/아카이브 수, **첨부 커버리지**(첨부 有 공고 %), 활성 룰/고객사 수, 마지막 재스코어링 시각(ai_flags.rescored_at 최댓값).
- **소스**: collect_cursor, bids(archived_at/ai_flags), rules, clients, bid_attachments.

### DFR-10 전역 필터 · 인터랙션 · 실시간
- **전역 필터**: 기간(기본 최근 30일)·주력분야(키워드그룹)·고객사 카테고리·상태(진행/임박/마감). 모든 위젯에 연동.
- **드릴다운**: 위젯의 값/막대/항목 클릭 → 해당 조건으로 S-04/S-05/S-07/S-06 이동(쿼리스트링 필터).
- **실시간**: bids Realtime 구독 → 변경 시 자동 갱신(FR-07 재사용). 수동 [새로고침].
- **역할별 뷰(선택)**: exec=요약/규모 중심, strategy/pm=액션/상세 중심(기본 전체 노출).

## 5. 신규 데이터 요구 (델타·추세 정확도)

### DFR-11 일일 스냅샷 (KPI 전일대비용, 선택)
- 전일 대비 델타·추세를 정확히 계산하려면 일 단위 집계 스냅샷이 필요.
- `dashboard_daily(snapshot_date, metrics jsonb)` 테이블 + 일 배치(수집 후) 1행 적재.
- 미도입 시: 델타는 notice_dt 기반 근사(신규 유입만), 추세는 원본 집계로 대체.

## 6. 데이터 소스 매핑 (요약)

| 위젯 | 주요 소스 |
|------|-----------|
| KPI/분포/추세/기관 | `bids`(score·ai_score·status·notice_dt·deadline_dt·est_price·order_org·demand_org·ai_summary·ai_flags·archived_at) |
| 고객사 | `clients`(category·is_priority) + bids 매칭 |
| 주력분야 | `bids.tags` ↔ `keyword_groups` |
| 액션/관심 | `watchlist`(analysis/proposal/decision) |
| AI 브리핑 | `daily_brief` |
| 데이터상태 | `collect_cursor`·`rules`·`bid_attachments` |
| 델타/추세 | `dashboard_daily`(신규, DFR-11) 또는 근사 |

> 모든 조회는 `archived_at is null` 필터 준수(정리된 공고 제외). RLS: active 전원 read.

## 7. 성능 / 갱신

- **조회 방식**: DB 집계(캐시성). AI 요약은 저장값 읽기(실시간 LLM 아님).
- **집계 최적화**: KPI/분포는 `count`/`head` 쿼리 또는 DB 뷰(`v_dashboard_stats`)로 1~수 회 호출. 대량 fetch 지양.
- **실시간**: bids Realtime → react-query invalidate. 폴백: 수동 새로고침.
- **권한**: active 전원(S-10). 관리 지표(파이프라인 상태)는 전원 열람 가능.

## 8. 기존(v1) 대비 강화 요약

| 영역 | v1(현행) | v2(강화) |
|------|----------|----------|
| KPI | 4종 | **10종 + 델타/스파크라인** |
| 액션 | 없음 | **마감임박·관심 진행·오늘의 액션(DFR-02)** |
| 고객사 | 없음 | **카테고리·Top·재수주·보건집중률(DFR-04)** |
| 품질 | 없음 | **점수/AI 분포·고득점 Top·AI커버리지(DFR-05)** |
| 추세 | 신규 14일 | **신규 + 향후 마감 예정(DFR-06)** |
| 브리핑/상태 | 없음 | **AI 데일리 브리핑·데이터 신선도(DFR-08/09)** |
| 인터랙션 | 정적 | **전역 필터 + 드릴다운 + 실시간(DFR-10)** |

## 10. 테스트 시나리오

1. 열자마자 KPI 10종 + 마감임박 + 주력분야 + 고객사 현황이 첫 화면에 표시.
2. KPI '고객사 공고' 클릭 → S-04가 고객사 공고로 필터되어 열림(드릴다운).
3. 기간 필터를 '최근 7일'로 → 전 위젯이 동기 갱신.
4. 신규 공고 수집(Realtime) → KPI·추세 자동 갱신.
5. daily_brief 당일 요약·Top 공고 표시, 미생성 시 안내.
