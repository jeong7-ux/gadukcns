# 수집 강화 방안 — AI 사업분류 정밀화 (검토·보고)

> 배경: "정보시스템(정보화) 감리·컨설팅과 무관한 사업이 수집됨" → 실측 진단 + 강화안 수립.

## 1. 현황 진단 (실측 · 적재 177건)

적재 = 감리 19 · 컨설팅 158. **대부분은 정분류**다 — ISP/ISMP/정보화전략계획/BPR 수립, 정보시스템 감리가 컨설팅·감리의 대다수를 차지하며 회사 도메인과 정확히 부합한다.

그러나 아래처럼 **무관 사업이 컨설팅으로 오분류**되어 섞였다(신뢰도·LLM 근거 첨부):

| 공고명(발췌) | 분류/신뢰도 | 실제 성격 | 판정 |
|---|---|---|---|
| 빅데이터 교과목 2단계 개선 | 컨설팅 0.75 | IT **교육과정** 개선 | ✗ 무관 |
| 신용카드 고객 빅데이터 기반 서베이 | 컨설팅 0.75 | 데이터 **서베이/조사** | ✗ 무관 |
| 기상 빅데이터·AI 산림기상정보 산출 방법 조사 | 컨설팅 0.85 | **연구/방법 조사** | ✗ 무관 |
| 게임 개발 인공지능 전환(AX) 지원사업 운영 | 컨설팅 0.65 | 지원사업 **운영** | ✗ 무관 |
| KIOST 연구선 빅데이터 통합관리시스템 고도화 | 컨설팅 0.75 | SI **고도화/구축** (ISP 아님) | △ 경계(구축) |
| K-CURE 암 빅데이터 구축 사업 | 컨설팅 0.75 | 데이터 **구축**(SI) | △ 경계(구축) |

- **오분류 규모: 약 5~8건(≈3~5%)**. 정밀도 자체는 나쁘지 않으나, 위 유형이 반복 유입될 구조적 허점이 있다.
- 공통점: **"빅데이터/인공지능/데이터"** 단어만으로 후보가 되고, LLM이 **"데이터/AI 컨설팅"** 정의를 넓게 적용해 교육·조사·연구·운영·단순구축까지 컨설팅으로 끌어올림.

## 2. 원인 (스테이지별)

| 스테이지 | 허점 |
|---|---|
| **Stage 1 사전선별** | keyword_groups **`AI/데이터`**(인공지능·빅데이터·데이터 구축·데이터 분석)가 후보 기준에 포함 → 교육/연구/조사/운영 공고까지 LLM 후보로 통과 |
| **Stage 2 LLM 프롬프트** | 컨설팅 정의에 **"데이터/AI 컨설팅"** 이 모호하게 포함 → "빅데이터 = 컨설팅"으로 과대해석. 구축/고도화/운영/교육/연구를 배제하는 지침 부재 |
| **Stage 3 임계값** | **KEEP=0.6** 이 낮아 0.65~0.75 경계건이 **보류(검수) 없이 자동 적재** |
| 입력 정보 | 메타(제목·기관)만 사용 → 사업 성격(자문 vs 구축) 판별에 정보 부족 |

## 3. 강화 방안 (다층 방어)

### (A) Stage 1 후보 기준 강화 — 코어 신호 중심
- 후보 자격을 **코어 컨설팅/감리 신호**로 한정: `감리 · ISP · ISMP · 정보화전략(계획) · 정보전략계획 · BPR · EA · 정보화 성과평가 · PMO · ISMS-P/정보보안 컨설팅 · 지능정보화 컨설팅`.
- **`AI/데이터` 단독 매칭은 후보 제외** → `AI/데이터` + 코어 신호(감리/ISP/전략/컨설팅/평가) 동시일 때만 후보.
- 구현: keyword_groups `AI/데이터` 그룹을 후보 트리거에서 분리(또는 코어 결합 조건 추가). rules 기반 base(감리 5·ISP/ISMP 4·전략 4)는 유지.

### (B) Stage 0 조건부 하드필터 — 무관 도메인 차단
- 다음 키워드가 있고 **코어 신호가 없으면 드롭**(코어 결합 시 예외 유지):
  `교과목 · 교육과정 · 커리큘럼 · 교재 · 콘텐츠 개발 · 서베이 · 설문 · 실태조사 · 여론조사 · 연구용역 · 산출 방법 · 지원사업 운영 · 운영 대행 · 홍보 · 행사`.
- 조건부이므로 "교육 플랫폼 **ISP** 수립"처럼 코어 신호가 있으면 유지된다.

### (C) Stage 2 프롬프트 정밀화 + few-shot
- 컨설팅 정의를 **자문/전략/계획/평가 성격으로 한정**: ISP/ISMP/정보화전략계획/BPR/EA/정보화 성과평가/PMO/정보보안(ISMS-P) 컨설팅.
- **명시적 해당없음 규칙 추가**:
  - 시스템 **구축·고도화·개발·운영·유지관리**만 있고 ISP/전략/감리 성격이 없으면 → 해당없음.
  - 데이터 **구축·분석·서베이·조사**, **교육/교과목/콘텐츠 개발**, **연구용역/방법 조사**, **지원사업 운영** → 해당없음.
  - "빅데이터/AI" 단어가 있어도 그 자체로 컨설팅이 아니며, **ISP/전략/자문/감리 성격일 때만** 컨설팅.
- **few-shot 예시** 삽입(정답: ISP 수립·정보시스템 감리 / 오답: 빅데이터 교과목 개선·데이터 구축·시스템 고도화·지원사업 운영).

### (D) Stage 3 임계값 상향
- **KEEP 0.6 → 0.8**, DROP 0.4 유지. 경계(0.4~0.8)는 **보류(검수 큐)** 로 이동 → 자동 적재 방지, 놓침은 사람이 확인.

### (E) 2차 검증(adversarial) — 적재 직전 재확인
- 적재 후보(감리/컨설팅)에 대해 저비용 1콜 재질의: *"이 사업은 IT 자문/전략/계획/평가/감리인가, 아니면 구축/개발/운영/교육/연구인가? 후자면 제외."*
- 다수 오분류(교육·조사·운영)를 최종 차단. LLM 호출은 Stage1 강화로 후보가 줄어 순증 최소.

### (F) 첨부 요지 반영 (후속)
- 메타만으로 애매한 건은 첨부(HWP/PDF) 추출 텍스트의 **사업 개요/과업 범위**를 분류 입력에 추가 → 자문 vs 구축 판별력 향상. (attachments 파이프라인 가동 후)

### (G) 소급 재분류 + 정리
- 강화 기준으로 **기존 177건 재분류**(classify_backfill 재실행) → 오분류(교육/조사/연구/운영)는 해당없음 처리·삭제, 경계(구축-only)는 보류로 이동.

## 4. 기대 효과

- **정밀도↑**: 교육·연구·조사·운영·단순구축 유입 차단. 회사 도메인(정보시스템 감리 + 정보화 전략/계획/평가/보안 컨설팅)에 집중.
- **재현율 보존**: 애매·경계는 드롭이 아닌 **보류·검수**로 → 기회 유실 방지.
- **비용 중립**: Stage1 강화로 LLM 후보↓, 2차 검증으로 후보당 +1콜 → 순증 최소. Stage0/1은 LLM 0.

## 5. 실행 단계 (구현 시)

1. `lib/collect/classify.ts` + `scripts/classify_backfill.mjs`: Stage0 조건부 제외어, Stage1 코어신호 후보기준, 프롬프트 정밀화+few-shot, 임계값(0.8/0.4), 2차 검증 추가.
2. keyword_groups `AI/데이터` 후보 트리거 분리(코어 결합 조건).
3. 소급 재분류(backfill) → 오분류 제거·보류 이동 → S-12 검수.
4. 검증: dry-run 표본으로 오분류 6종이 해당없음/보류로 재판정되는지 확인 + 정분류(ISP/감리) 유지 확인.

## 부록. 판정 기준 요약(운영자용)

- **감리** = 정보시스템(정보화) 감리 — 정보시스템감리사/감리원 배치·감리대가, 정보화사업 제3자/PMO 감리. (건설·전기·소방 감리 제외)
- **컨설팅** = 정보화 **자문/전략/계획/평가**: ISP·ISMP·정보화전략계획·BPR·EA·정보화 성과평가·PMO·정보보안(ISMS-P) 컨설팅.
- **해당없음** = 시스템 구축·고도화·개발·운영·유지관리(자문/전략/감리 없음), 데이터 구축·분석·조사, 교육·콘텐츠, 연구용역, 지원사업 운영, 물품·건설·일반용역.
