# 기능검토 보고서 — 우선순위 발주기관 정렬 + 배경색

> 작성일: 2026-07-12 · 대상: S-04(입찰 목록)·S-05(키워드그룹 검색) · 상태: **검토(설계) — 미구현**
> 요청: 우리 주력사업(정보시스템 감리·컨설팅, 주로 보건·복지 분야) 기준으로 **자주 거래하는 발주기관**(예: 질병관리청·국립보건연구원·보건복지부·식품의약품안전처 등)을 **우선 정렬**하고 **배경색**으로 강조.

## 1. 결론 — **가능합니다** ✅

두 요구(우선 정렬·배경색) 모두 구현 가능하며, **시스템에 이미 있는 메커니즘을 재사용**하면 신규 부담이 작습니다.

## 2. 이미 있는 것 — `rules(type='org')` = 관심 발주기관 가중

스코어링 설계(FR-05, 7.2)에 발주기관 가중이 이미 정의돼 있습니다:
```
score = Σ(keyword/contract weight) + agency_bonus(관심 발주기관) − exclude_penalty
```
- `rules` 테이블은 `type in ('keyword','org','exclude','contract')` 지원.
- **`type='org'`** 룰: 패턴이 공고의 발주기관/수요기관(`order_org`+`demand_org`)에 포함되면 **agency_bonus**를 점수에 가산.
- S-12(스코어링 규칙) UI가 이미 **"발주기관" 유형**을 지원(추가/가중치/활성 토글).
- 재스코어링(`rescore_bids`)·`ai.mjs`·정리 함수 모두 org 룰을 반영.

→ 즉 **"질병관리청 등을 발주기관(org) 룰로 등록 + 큰 가중치"** 만으로 우선순위 점수가 붙습니다. 별도 테이블 불필요.

## 3. 우선순위 발주기관 정의 — 2안

| 안 | 방법 | 장점 | 단점 |
|----|------|------|------|
| **A. org 룰 재사용 (권장)** | S-12에서 질병관리청·국립보건연구원·보건복지부·식약처 등을 `발주기관` 룰로 추가(weight 예: 10) | 기존 인프라·UI 재사용, 점수/정렬/색 일괄 연동, admin이 직접 관리 | 점수에 섞임(원치 않으면 A' 참고) |
| A'. org 룰 + 전용 태그 | org 룰에 표식(예: weight≥10)을 "우선기관"으로 간주 | 점수와 우선표시 분리 명확 | 규칙 약간 복잡 |
| B. 전용 `priority_orgs` 테이블 | 별도 목록·관리 화면 | 점수와 완전 분리 | 신규 테이블·UI·DDL 필요(적용 부담) |

> **권장: A안.** "자주 거래 = 관심 발주기관 가중"이라는 스펙 의도와 정확히 일치.

## 4. 우선 정렬 — 구현 방식

현재 S-04/S-05 목록은 `notice_dt desc`(최신순) 정렬(`lib/queries/bids.ts`).

| 정렬 방식 | 구현 | 특성 |
|-----------|------|------|
| **점수 우선(권장)** | org 룰 가중 → 재스코어링 → `.order('score', desc).order('notice_dt', desc)` | 우선기관+고관련 공고가 상단. DB 레벨 정렬(200건 한계 내 정확) |
| 우선기관 플래그 우선 | 우선기관 목록을 받아 `order_org` 매칭 공고를 앞으로(클라이언트 정렬) | 점수와 독립적 "기관 먼저". 단, 200건 페치 범위 내에서만 |
| 하이브리드 | 우선기관 여부(1순위) → 점수(2순위) → 마감/최신(3순위) | 가장 직관적. 우선기관 목록 필요 |

- **DB 레벨(점수 우선)** 이 가장 견고: 우선기관 공고가 점수로 올라가 페치 범위(200) 안에 확실히 포함.
- 순수 "기관 우선"을 원하면 하이브리드: 우선기관 목록으로 `order_org` 매칭을 최상단 그룹으로.

## 5. 배경색(강조) — 구현 방식

- **어디에**: S-04/S-05 공통 카드(`BidCard`). 우선 발주기관 공고 카드에 **연한 배경 틴트**.
- **판정**: 우선기관 목록(org 룰의 패턴들)을 조회 → `bid.order_org`/`demand_org`에 포함되면 우선기관.
  - 조회 1회(`rules?type=eq.org&is_active=eq.true`) → 목록 캐시 → 카드에서 substring 매칭.
  - 또는 재스코어링 시 org 매칭 패턴이 `tags`에 들어가므로, org 룰 패턴 집합과 교집합으로 판정.
- **색상(디자인 토큰)**: 하드코딩 금지. 신규 토큰 제안 `--color-priority-bg`(라이트·다크 각각) 또는 기존 `accent` 틴트 재사용(`bg-accent/5` + 좌측 강조 바 `border-l-2 border-accent`).
  - 라이트: 예) 연한 블루 `#EFF4FB`
  - 다크 모드 대응 시 별도 값(현재 앱은 라이트 기준).
  - 가독성: 텍스트 대비 유지되는 아주 옅은 톤 권장(#1F497D 텍스트가 잘 보이는 수준).
- **추가 표식(선택)**: 카드에 `우선기관` 핀 배지(예: `⭐ 우선` pill) 병행하면 색약 사용자 접근성↑.

## 6. 적용 범위

- S-04·S-05는 **동일한 `fetchBids` + `BidCard`** 사용 → 한 번 수정으로 양쪽 반영(첨부 배지·상태 pill과 동일 패턴).
- 통계(S-10)에도 "우선 발주기관 공고 수" 지표 추가 가능(선택).

## 7. 작업 범위 / 영향 (A안 + 점수우선 + 배경색 기준)

| 대상 | 변경 |
|------|------|
| 데이터 | S-12에서 발주기관(org) 룰 등록(질병관리청 등) + 재스코어링 (코드변경 없음) |
| `lib/queries/bids.ts` | 정렬 `score desc, notice_dt desc`로 변경 + 우선기관 목록 조회·부착(`is_priority_org`) |
| `components/bids/BidCard.tsx` | 우선기관이면 배경 틴트 + (선택) `⭐ 우선` 배지 |
| `tailwind.config.ts`/`globals.css` | `--color-priority-bg` 토큰 추가(선택) |
| DDL/함수 | **불필요**(org 룰은 기존 스키마·함수가 이미 처리) |

→ **DDL 없이, 프론트 2파일 + S-12 데이터 입력**으로 완성. 부담 작음.

## 8. 엣지 케이스 / 고려사항

- 발주기관 표기 흔들림(예: "조달청 부산지방조달청"이 실제 발주, 수요기관이 질병관리청) → **`demand_org`도 함께 매칭**해야 실수요기관 기준 우선순위가 정확.
- 부분매칭 오탐: "○○보건소"가 "보건"에 걸릴 수 있음 → 패턴을 기관 정식명칭으로(예: '질병관리청', '식품의약품안전처') 구체화.
- 점수우선 정렬 시 키워드 점수와 섞임 → 순수 기관우선이 필요하면 하이브리드(기관 플래그 1순위) 채택.
- 다크모드: 배경 틴트 토큰을 라이트/다크 각각 정의.

## 9. 권고안

1. **A안(org 룰 재사용) + 점수우선 정렬 + 배경 틴트(+⭐배지)** 로 구현.
2. 초기 우선기관 시드(예): 질병관리청·국립보건연구원·보건복지부·식품의약품안전처·국민건강보험공단·건강보험심사평가원·한국보건산업진흥원 등 보건·복지 분야. weight 10 권장.
3. **`demand_org`(수요기관)까지 매칭** 하여 조달청 대행 공고도 실수요기관 기준으로 우선순위 부여.
4. 색은 디자인 토큰(`--color-priority-bg`)로 중앙화, 접근성 위해 배지 병행.

## 10. 예상 결과

- 보건·복지 발주기관 공고가 목록 상단에 모이고, 카드가 연한 배경+⭐로 즉시 식별.
- S-12에서 기관 추가/가중치 조정 → 재스코어링 → 즉시 반영(운영 유연).
