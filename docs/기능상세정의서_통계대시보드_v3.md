# 통계 대시보드(S-10) 기능상세정의서 v3.0 — 현실 기반 재설계

> 작성일: 2026-07-13 · 대상 화면: **S-10 통계 대시보드** (FR-11 · v2 개정)
> 핵심: **고객사 · 스코어링(4~9) · 마감순위** 3축으로 재구성. "많이 보여주기"(v2) → **"실데이터에 맞게 정확히 보여주기"(v3)**.
> 짝 문서: `UIUX설계서_통계대시보드_v3.md` (레이아웃·가독성 규칙)

## 1. 왜 v3인가 — 실측 데이터 프로파일 (2026-07-13)

v2는 10-KPI·9-위젯 "커맨드센터"를 지향했으나, 실제 운영 데이터 규모와 어긋나는 부분이 확인됐다.

| 실측 항목 | 값 | v3 설계 반영 |
|---|---|---|
| 노출 공고(미아카이브·마감전) | **217건** | 마감 제외가 전 위젯 기본 (v2엔 없던 규칙) |
| 주력사업 공고(주력점수≥4) | **61건** | 대시보드의 실질 모수 — KPI 부풀림 제거 |
| 주력점수 분포 | **4점: 35 · 5~9점: 26 · 10점+: 0** | **점수 밴드는 4~9가 전부** → 2밴드(4=관심 / 5~9=우량)로 단순화. v2의 10~14/15+/20+ 버킷은 항상 0 → 삭제 |
| 고객사 매칭 공고 | **19건** | 독립 트랙으로 승격 |
| 고객사 ∩ 주력≥4 | **2건** | **고객사 트랙과 주력 트랙은 사실상 서로 다른 집합** → 교집합 강조가 아니라 **듀얼 트랙** 분리 표시 |
| D-day 분포(주력, 마감일 有) | D0~3: 26 · D4~7: 0 · D8~14: 5 · D15+: 4 · 미정: 26 | 마감순위 리스트가 핵심 축. **마감일 미정(26건)이 절반** → 미정 그룹 명시 표기 필수 |
| AI 요약 커버리지 | 36/61 (59%) | 품질 지표 유지 |
| 관심목록 | 1건 | v2의 "관심 진행 현황 보드"는 과설계 → 카운트 배지로 축소 |

**결론**: 실데이터에서 의사결정에 쓰이는 질문은 3가지다.
1. **지금(마감순으로) 무엇을 잡아야 하나** → 마감순위 액션 보드
2. **고객사에서 무엇이 나왔나** → 고객사 트랙
3. **주력 기회가 얼마나·어떤 품질로 있나** → 스코어링 4~9 현황

## 2. 설계 원칙 (v3)

1. **3축 우선**: 마감순위(시간) × 고객사(관계) × 주력점수 4~9(적합도). 이 3축 밖 정보는 하단 보조 영역으로.
2. **정직한 숫자**: 전 위젯 모수 = **미아카이브 + 마감전(deadline ≥ 오늘 or 미정)**. 총점(가산 포함)이 아닌 **주력점수(base−exclude)** 기준. S-04·S-05와 동일 규칙(세션 표준).
3. **듀얼 트랙**: 고객사 공고(19)와 주력 공고(61)는 교집합 2건뿐 → 하나의 리스트에 섞지 않고 **트랙별 분리**. 교집합 발생 시 ⭐+주력 배지 동시 표기로 최상위 강조.
4. **2밴드 스코어**: `4점 = 관심(주력 키워드 1개 매칭)` / `5~9점 = 우량(복수·고가중 매칭)`. 10+ 버킷은 데이터가 생기기 전까지 표시하지 않는다(빈 축 금지).
5. **마감 미정도 정보다**: 주력 공고의 43%가 마감일 미정 → "미정 N건" 그룹을 숨기지 않고 별도 행으로 노출.
6. **드릴다운 유지**: 모든 리스트 항목 → S-06 상세, KPI → S-04 목록.

## 4. 위젯별 기능정의 (DFR3-01 ~ DFR3-07)

### DFR3-01 KPI 밴드 (6타일 — v2 10타일에서 축소)

| # | 타일 | 정의(모수: 미아카이브·마감전) | 클릭 |
|---|---|---|---|
| 1 | **주력 공고** | 주력점수 ≥ 4 | S-04 |
| 2 | **우량(5~9)** | 주력점수 5~9 | S-04 |
| 3 | **고객사 공고** | 발주/수요기관 = clients 매칭 (점수 무관) | S-04 |
| 4 | **임박 D0~3** | 주력≥4 ∧ dday 0~3 | B 보드 스크롤 |
| 5 | **총 추정가(억)** | Σ est_price / 1e8 (주력≥4) | — |
| 6 | **AI 커버리지** | ai_summary 有 / 주력≥4 (%) | — |

- 제거(v2 대비): 진행중(마감전 모수와 중복), 관심목록(1건 — B 보드 배지로 이동), 신규24h(0건 빈번 — F 추세로 대체), 평균점수(2밴드 표시로 대체).
- 델타/스파크라인(DFR-11 스냅샷 의존)은 **보류** — dashboard_daily 도입 시 재개.

### DFR3-02 마감순위 액션 보드 (B — 첫 화면 좌측, 최우선)

- **모수**: 주력점수 ≥ 4 ∧ 마감전. **정렬: ① D-day 오름차순 → ② 고객사(⭐) → ③ 주력점수 내림차순.**
  (S-04 목록 정렬은 고객사 우선이지만, 이 보드의 존재 이유가 "마감순위"이므로 시간축을 1순위로 둔다.)
- **구간 그룹 헤더**: `D0~3(urgent)` → `D4~7(soon)` → `D8~14(near)` → `D15+(far)` → `마감 미정`. 구간별 건수 병기. 빈 구간은 헤더째 생략(빈 축 금지).
- **행 구성**: D-day pill(3구간 토큰) · (발주기관) · 사업명 · ⭐(고객사) · 주력점수 배지 · 추정가(있으면).
- **표시 수**: 기본 D0~7 전체 + 이후 구간 접기(펼치기 토글). 최대 20행.
- **하단 배지 줄**: 오늘 마감 N · 관심목록 N(→S-07) · 마감 미정 N.
- 드릴다운: 행 → S-06.

### DFR3-03 고객사 트랙 (C — 첫 화면 우측)

- **모수**: 고객사 매칭 공고(마감전) — **점수 무관**(고객사∩주력=2건뿐이므로 점수로 거르면 트랙이 소멸).
- **상단 요약 스트립**: 카테고리 칩(중앙부처 N · 지자체 N · 의료 N · 공공 N · 금융 N — 0인 칩 생략).
- **공고 리스트**: 마감순 정렬, 고객사명 강조·사업명·D-day pill·주력점수(≥4면 배지). 최대 10행.
- **재수주 신호**: 동일 고객사 공고 2건 이상이면 카운트 표기(`분당서울대병원 ×3`).
- 드릴다운: 칩 → S-04(기관 필터), 행 → S-06.

### DFR3-04 스코어링 현황 (D — 2밴드)

- **분포**: 가로 스택바 1개 — `4점(관심) 35 ▏ 5~9점(우량) 26` (버킷 차트 대체. 10+ 밴드는 발생 시 자동 추가).
- **우량 Top 8**: 주력점수 내림차순(동점은 마감 임박순) — 사업명·주력점수·⭐·D-day. v2 Top 5 → 8 확대(모수 61이라 8이 상위 13%).
- **AI 커버리지 게이지**: 요약 有/주력 공고(%). 클릭 시 미요약 공고 리스트(온디맨드 요약 유도, §20 기능 연계).
- 드릴다운: 행 → S-06.

### DFR3-05 마감 캘린더 미니 (E)

- 다음 14일 일자별 마감 건수 막대(주력≥4). D-day 3구간 색. 막대 클릭 → S-08 해당 일자.
- 14일 내 마감 0건이면 "14일 내 마감 없음 · 미정 N건" 문구로 대체(빈 차트 금지).

### DFR3-06 보조 영역 (F — 접이식/하단)

- **유입 추세**: 최근 21일 일자별 신규(주력≥4) 라인. v2 유지.
- **발주기관 Top 5**: 주력≥4 기준 가로바(v2 Top 8 → 5 축소, 롱테일 잘라 가독 확보).
- **AI 데일리 브리핑**: daily_brief 당일 요약. v2 유지.
- **데이터 상태**: 마지막 수집·재스코어링·노출/아카이브·활성 룰·고객사 수. v2 유지.
- v2의 수요기관 Top 8·고객사vs일반 도넛·점수 히스토그램(다버킷)·관심 진행 보드는 **삭제**(중복·빈 데이터).

### DFR3-07 공통 규칙 (필터·정렬·실시간)

- **공통 모수 필터**: `archived_at is null` ∧ (`deadline_dt ≥ today` ∨ `deadline_dt is null`). 위젯별 추가 조건은 위 정의대로.
- **주력점수 정의**: `ai_flags.score_breakdown.base − exclude`, breakdown 없으면 총점 폴백 (lib/queries 공통 헬퍼로 단일화 — stats.ts·bids.ts 중복 제거).
- **실시간**: bids Realtime → invalidate(기존 재사용). 수동 새로고침 버튼 유지.
- **권한**: active 전원 read (기존 동일).

| 영역 | v2 | v3 | 사유(실측) |

## 8. 테스트 시나리오

1. 열자마자 A(6 KPI)+B(마감보드)+C(고객사 트랙)가 첫 화면(above-the-fold, 1440px)에 보인다.
2. B 보드: D0~3 그룹이 최상단, 각 행 D-day pill 색이 3구간 토큰과 일치, 마감 미정 그룹이 맨 아래 접힘.
3. C 트랙: 고객사 19건이 점수와 무관하게 표시되고, 주력≥4인 2건에는 주력 배지 동시 표기.
4. D 스코어링: 스택바 합계 = KPI '주력 공고' 값과 일치. 10+ 밴드 미표시.
5. KPI '우량(5~9)' 클릭 → S-04로 이동.
6. 신규 수집(Realtime) → 전 위젯 자동 갱신.
7. 다크모드 토글 → 전 위젯 토큰 색 즉시 반영(흰 배경 잔존 없음).
