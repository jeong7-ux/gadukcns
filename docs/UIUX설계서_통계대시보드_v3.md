# 통계 대시보드(S-10) UI/UX 설계서 v3.0 — 가독성 중심

> 작성일: 2026-07-13 · 짝 문서: `기능상세정의서_통계대시보드_v3.md` (데이터·규칙 정의)
> 설계 목표: **3초 안에 "오늘 뭘 잡아야 하나"가 읽히는 화면**. 장식이 아닌 정보 위계로 가독성을 만든다.

## 1. 가독성 원칙 (전 위젯 공통)

| # | 원칙 | 구체 규칙 |
|---|---|---|
| R1 | **숫자가 주인공** | KPI 숫자 `text-2xl font-bold`, 라벨 `text-[11px] text-subtle`. 숫자:라벨 크기비 ≥ 2:1 |
| R2 | **색은 의미로만** | 색상 의미 고정표(§2) 밖의 장식색 금지. 한 위젯 내 의미색 최대 3종 |
| R3 | **빈 축 금지** | 0건 구간·빈 차트는 그리지 않고 문구로 대체 ("14일 내 마감 없음") |
| R4 | **행 스캔 최적화** | 리스트 행: 좌측 고정폭 신호(pill/배지) → 중앙 가변 텍스트(truncate) → 우측 고정폭 수치. 행높이 36~40px |
| R5 | **그룹 헤더로 청킹** | 긴 리스트는 5±2행 단위 구간 헤더(D0~3/D4~7…)로 분절 |
| R6 | **본문 대비** | 본문 `text-text`(AA 4.5:1↑), 보조 `text-subtle`, 비활성만 opacity. 회색 3단계 초과 금지 |
| R7 | **차트 최소화** | 21일 추세·분포 스택바 외 차트 지양. 20건 미만 데이터는 차트보다 리스트 |
| R8 | **다크모드 동급** | 모든 색은 CSS 변수 토큰. recharts Tooltip/축/cursor 포함(§19·26 기수정 패턴 준수) |

## 2. 디자인 토큰 · 색상 의미 고정표

기존 토큰(globals.css) 재사용 — 신규 색 없음.

| 의미 | 토큰 | 라이트 | 용도 |
|---|---|---|---|
| 브랜드/주력 | `--color-primary` | #1F497D | 주력점수 배지, KPI 주력 계열 |
| 고객사/링크 | `--color-accent` | #2563EB | ⭐ 고객사 표시, 드릴다운 hover |
| 긴급 D0~3 | `--color-dday-urgent` | #DC2626 | D-day pill·임박 KPI |
| 주의 D4~6 | `--color-dday-soon` | #EA580C | D-day pill |
| 근접 D7~9 | `--color-dday-near` | #EAB308 | D-day pill |
| 여유/미정 | `--color-dday-far` | #64748B | D-day pill·미정 그룹 |
| 정상/완료 | `--color-success` | #16A34A | AI 커버리지 충족, 실시간 연결 |
| 본문/보조 | `--color-text` / `--color-text-subtle` | — | R6 |
| 면/경계 | `--color-surface` / `--color-border` | — | Card·divider |

**규칙**: D-day 색은 `lib/design/dday.ts` 3구간과 1:1 — 다른 의미로 재사용 금지. ⭐은 항상 accent 단색(그라데이션·애니메이션 금지).

## 4. 위젯별 UI 스펙

### A. KPI 타일 (6개)

```
┌──────────────┐   라벨: text-[11px] text-subtle, 1줄
│ 주력 공고      │   숫자: text-2xl font-bold + 의미색(§2)
│ 61           │   단위(억/%)는 text-sm으로 숫자에 병기
└──────────────┘   클릭 가능 타일: hover:-translate-y-0.5 (기존 Kpi 패턴)
```

- 색: 주력=primary, 우량=primary, 고객사=accent, 임박=dday-urgent, 추정가=text, AI=success(≥70%)/subtle(미만).
- 모바일: `grid-cols-2`, 태블릿 `grid-cols-3`.

### B. 마감순위 액션 보드 — 화면의 심장

**행 템플릿** (R4: 신호─텍스트─수치 3열 스캔 구조, h-9):

```
[D-2]  (발주기관) 사업명사업명사업명…            ⭐ [주5] 3.2억
 └48px └text-subtle └text-text truncate      └우측 고정폭 정렬
```

- **D-day pill**: `DDAY_PILL_CLASS` 3구간 그대로. 마감 미정 그룹 행은 pill 대신 `[미정]`(far 톤).
- **주력점수 배지** `주N`: `bg-primary/10 text-primary rounded px-1.5 text-[11px] font-semibold`.
- **⭐**: 고객사 매칭 시에만, 사업명 뒤 고정 위치(행마다 좌우 요동 금지).
- **그룹 헤더**: `text-xs font-semibold text-subtle uppercase` + 건수 + 접기 토글(▾/▸). D0~7은 기본 펼침, 이후 접힘.
- **빈 구간**: 헤더째 렌더 생략(R3).
- **하단 배지 줄**: 기존 Pill 컴포넌트 — `danger 오늘 마감 N` · `accent 관심 N` · `muted 마감 미정 N`.

### C. 고객사 트랙

- **카테고리 칩 스트립**: `rounded-full bg-bg ring-1 ring-border px-2.5 py-1 text-xs` — `중앙부처 6`. 0건 칩 생략. 클릭 → S-04 기관 필터.
- **고객사 그룹핑 리스트**: 같은 고객사 공고는 묶고 `고객사명 ×N`(font-medium text-accent) 헤더 아래 들여쓰기 행. 각 행: 사업명 truncate + D-day pill + (주력≥4면) 주력 배지.
- 10행 초과 시 "전체 보기 →"(S-04 고객사 정렬이 이미 고객사 우선이므로 그대로 연결).

### D. 스코어링 현황

- **2밴드 스택바** (차트 라이브러리 불필요 — div 스택으로 구현):

```
관심(4) 35 ▓▓▓▓▓▓▓▓▓▓▓▓░░░░░░░░ 26 우량(5~9)
           └ bg-primary/40      └ bg-primary
```

  좌우 라벨+수치 밖으로, 바 위 텍스트 금지(대비 문제). 높이 20px, rounded.
- **우량 Top 8 리스트**: B 보드와 동일 행 템플릿(일관성) — 단 D-day 대신 주력점수가 우측 주 수치.
- **AI 커버리지**: 가로 게이지 + `59%` + `[미요약 25건]` 버튼(text-xs ring-1). 70% 이상 success, 미만 subtle.

### E. 마감 캘린더 미니

- 14열 미니 막대(높이 64px), 막대색 = 해당일 D-day 구간색, 오늘 눈금 강조.
- 축 라벨은 3~4개만(07.14 · 07.18 · 07.22 · 07.27) — 전 일자 라벨 금지(R7).
- 데이터 0이면 렌더 생략 + "14일 내 마감 없음 · 미정 26건" 문구(R3).

### F. 보조 영역

- 기본 접힘(모바일)/펼침(데스크톱) `<details>` 스타일 섹션 헤더.
- 유입 추세: 기존 LineChart 유지 + tt 토큰 스타일(§26 패턴) 필수.
- 발주기관 Top 5: 가로바, 기관명 12자 truncate.
- AI 브리핑·데이터 상태: 기존 컴포넌트 재사용.

## 5. 타이포그래피 스케일 (전 화면 5단계 고정)

| 단계 | 클래스 | 용도 |
|---|---|---|
| XL | `text-2xl font-bold` | KPI 숫자 |
| L | `text-sm font-semibold` | 카드 제목(CardHeader 기존) |
| M | `text-sm` | 리스트 본문(사업명) |
| S | `text-xs` | 보조 정보·칩·버튼 |
| XS | `text-[11px]` | pill·배지·라벨 |

5단계 밖 크기 사용 금지 — 크기 종류가 곧 시각 노이즈다.

## 6. 인터랙션

| 대상 | 동작 |
|---|---|
| KPI 타일 | hover 부상 + 클릭 → S-04 (기존 패턴) |
| 리스트 행 | hover `bg-bg` + 클릭 → S-06 상세 |
| 그룹 헤더 ▾/▸ | 접기/펼치기 (로컬 상태, 저장 불필요) |
| 카테고리 칩 | 클릭 → S-04 기관 필터 |
| 미요약 N건 | 클릭 → 미요약 리스트 모달 or S-04 (온디맨드 요약 연계) |
| 실시간 | bids Realtime → 자동 갱신, 헤더 ● 상태점(success/muted) |

로딩: 위젯별 스켈레톤(회색 블록) — 전체 화면 "불러오는 중…" 단일 문구 지양(위젯 단위 렌더).

## 7. 반응형

| 구간 | A KPI | B/C | D/E | F |
|---|---|---|---|---|
| ≥1280 (lg) | 6열 | 3:2 2열 | 2열 | 2열 |

가로 스크롤 전면 금지 — 넘치는 표는 카드 내 `overflow-x-auto`.

## 8. 다크모드

- 전 색상 CSS 변수(§2) — 하드코딩 HEX 금지.
- recharts 잔존 차트(추세·기관Top)는 §26 기수정 패턴(tt contentStyle·cursor·tick fill) 준수.
- 2밴드 스택바·게이지는 div 구현이라 토큰 자동 적용.
- 검증 항목: 토글 즉시 Tooltip·pill·게이지·막대 전부 반전(흰 잔존 0).

## 9. 접근성

- D-day 구간을 색으로만 구분하지 않는다 — pill 안에 항상 `D-2`/`미정` 텍스트.
- ⭐에 `aria-label="고객사 공고"`. 접기 토글에 `aria-expanded`.
- 리스트 행은 `<a>`(Link) — 키보드 포커스 링 유지.
- 대비: subtle 텍스트도 AA(4.5:1) 통과 토큰만 사용.
