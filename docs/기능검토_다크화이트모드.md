# 기능검토 보고서 — 화면 다크/화이트 모드

> 작성일: 2026-07-12 · 대상: 전 화면(S-01~S-14) · 상태: 검토(미구현)

## 1. 결론 — **가능하며, 난이도 낮음** ✅

이 프로젝트는 처음부터 **"색상 하드코딩 금지, 디자인 토큰 중앙화"** 원칙으로 만들어져, 다크모드 도입에 **이미 90% 준비**돼 있다. CSS 변수 값 세트 하나 추가 + 토글 + 소수의 하드코딩 색상 정리로 완성된다.

## 2. 현재 구조 (다크모드에 유리)

- **색상 단일 원천**: `app/globals.css`의 `:root`에 전 색상이 CSS 변수로 정의(`--color-bg/surface/text/border/primary/…`).
- **Tailwind 토큰 매핑**: `tailwind.config.ts`가 이 변수를 `bg/surface/text/border/primary…` 클래스로 노출. 컴포넌트는 대부분 이 토큰 클래스만 사용(`bg-surface`, `text-text`, `border-border` 등).
- **루트 레이아웃**: `app/layout.tsx` → `<html lang="ko"><body><Providers>` — 테마 속성/토글을 얹기 좋은 단일 진입점.
- **차트(recharts)**: `color("--color-primary", fallback)` 헬퍼로 CSS 변수를 읽어 색을 주입 → 변수만 바꾸면 **차트도 자동 적응**.

→ 즉 `--color-*` 변수에 **다크 값 세트**만 정의하면 전 화면이 통째로 전환된다.

## 3. 걸림돌 — 하드코딩 색상(소수·국소)

토큰을 우회한 하드코딩 HEX가 4개 파일에만 존재한다(대부분 의미색이라 다크에서도 무난하지만, 완성도를 위해 토큰화 권장):

| 파일 | 색상 | 처리 |
|------|------|------|
| `dashboard/stats/page.tsx` | 그리드선 `#e2e8f0` ×6 | **`var(--color-border)`로 토큰화 필요**(다크에서 밝은 회색선 튐) |
| `dashboard/stats/page.tsx` | `color()` 폴백/팔레트(#1F497D 등) | 폴백은 CSS변수 우선이라 자동 적응. 팔레트 보라/청록은 다크에서도 가독 OK |
| `calendar/page.tsx` | 이벤트색 `#2563EB`/`#DC2626` | 의미색(accent/danger) — 토큰 참조로 정리 권장 |
| `ui/StatusPill.tsx` | `#16A34A`/`#DC2626` | 상태색 — 다크에서도 유효, 선택적 토큰화 |
| `lib/design/dday.ts` | D-day 4색 | 임박도 색 — 다크에서도 유효, 유지 가능 |

> 핵심 구조색(bg/surface/text/border/subtle)은 **전부 토큰**이라 걸림돌 없음. 하드코딩은 의미색 위주 소수.

## 4. 구현 방식

### 4-1. 테마 전략 (권장: 수동 토글 + 시스템 기본)
- `tailwind.config.ts`에 `darkMode: ["class", '[data-theme="dark"]']` 지정.
- `<html data-theme="dark|light">` 속성으로 전환. 기본값은 **시스템 설정(prefers-color-scheme)** 을 따르고, 사용자가 토글로 override.

### 4-2. 다크 값 세트 (globals.css)
```css
:root[data-theme="dark"] {
  --color-bg: #0f172a;        /* slate-900 */
  --color-surface: #1e293b;   /* slate-800 */
  --color-text: #f1f5f9;      /* slate-100 */
  --color-text-subtle: #94a3b8;
  --color-border: #334155;    /* slate-700 */
  --color-primary: #3b82f6;   /* 네이비는 다크에서 대비 위해 밝게 */
  --color-accent: #60a5fa;
  --color-priority-bg: #172554; /* 우선 고객사 배경 다크 틴트 */
  /* success/danger/dday는 다크에서도 대비 충분 → 유지 또는 미세조정 */
}
```
- (선택) 시스템 자동: `@media (prefers-color-scheme: dark) { :root:not([data-theme="light"]) { …다크값… } }`.

### 4-3. 토글 UI + 지속성
- 헤더(우상단, 로그아웃 옆)에 **테마 토글 버튼**(☀/🌙). `ThemeToggle` 컴포넌트.
- 선택값을 `localStorage['theme']`에 저장, `<html data-theme>`에 반영.

### 4-4. FOUC(깜빡임) 방지
- SSR→hydration 사이 라이트로 번쩍이는 것 방지 위해, `<head>`에 **인라인 스크립트**로 초기 테마를 즉시 적용:
```html
<script>document.documentElement.dataset.theme =
  localStorage.theme || (matchMedia('(prefers-color-scheme: dark)').matches ? 'dark':'light')</script>
```

## 5. 작업 범위 (DDL/백엔드 무관 — 프론트만)

| 대상 | 작업 |
|------|------|
| `app/globals.css` | `:root[data-theme="dark"]` 다크 변수 세트 추가 |
| `tailwind.config.ts` | `darkMode` 전략 지정 |
| `app/layout.tsx` | FOUC 방지 인라인 스크립트 + `<html>` 속성 |
| `components/ui/ThemeToggle.tsx`(신규) | 토글 버튼 + localStorage |
| 헤더/셸 레이아웃 | 토글 버튼 배치 |
| 하드코딩 색 정리 | stats 그리드선 → `var(--color-border)`, calendar 이벤트색 토큰화 (약 8곳) |

→ **신규 1개 컴포넌트 + CSS/설정 수정 + 하드코딩 8곳 정리.** 반나절 수준.

## 6. 접근성 / 고려사항

- **대비(WCAG AA)**: 다크 surface(#1e293b) 위 text(#f1f5f9) 대비 충분. 브랜드 네이비(#1F497D)는 다크 배경 대비가 낮아 **primary를 밝게(#3b82f6)** 조정 필요.
- **차트**: recharts는 CSS 변수를 읽어 자동 적응(그리드선만 토큰화하면 완결). 도넛/막대 팔레트는 다크에서도 채도 충분.
- **상태·D-day 색**: success/danger/urgent 등 의미색은 다크에서도 유효(필요 시 채도만 미세조정).
- **FullCalendar**: `--fc-border-color`가 이미 `var(--color-border)` 참조 → 자동 적응. 이벤트색만 토큰화.
- **이미지/로고**: 텍스트 로고라 무관.
- **인쇄/PDF**: 해당 없음.

## 7. 권고안

1. **수동 토글 + 시스템 기본** 방식으로 도입(사용자 제어 + 첫 방문 자동).
2. `:root[data-theme="dark"]` 변수 세트 + `ThemeToggle` + FOUC 스크립트.
3. 하드코딩 색 8곳(특히 stats 그리드선)을 토큰화하여 완성도 확보.
4. primary만 다크용으로 밝게 조정(대비), 나머지 의미색은 유지.
